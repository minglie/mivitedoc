# 工程路径 vs 科研路径 对比表

| 维度           | 工程路径 🛠️                              | 科研路径 🔬                          |
|----------------|----------------------------------------|-------------------------------------|
| **成长方式**   | 从中级 → 高级 → 架构师，逐步堆经验      | 本科/硕士后直接科研，读博走前沿方向  |
| **技能结构**   | 基础扎实，熟悉 AXI/DMA/驱动，能动手调试 | 偏理论与设计，可能对底层细节生疏     |
| **中级能力**   | 很强：写 RTL、调 DMA、做中断都得心应手 | 偏弱：可能只会用现成 IP，不擅长 debug |
| **高级能力**   | 很强：复杂 DMA、PR、多核系统都能搞定   | 中等：理解原理，但动手较少           |
| **究极能力**   | 强：能做架构设计，同时兼顾落地性       | 中等：框架前瞻，但落地难             |
| **终极能力**   | 中等：科研意识有限，创新点不多         | 很强：Chiplet、SNN、异构计算探索    |
| **思维方式**   | 工程驱动：问题导向，先解决 bug 再优化   | 科研驱动：假设导向，先论文模型再验证 |
| **职业发展**   | 工程专家 / 技术负责人 / 架构师         | 教授 / 博士后 / 研究所 / 前沿实验室 |
| **养家指数**   | ⭐⭐⭐⭐⭐ 稳定高薪（大厂/芯片公司）        | ⭐⭐～⭐⭐⭐ 不稳定（科研资助/论文驱动） |
| **风险点**     | 技术更新慢时容易瓶颈                   | 缺乏工程经验，产业落地困难           |
